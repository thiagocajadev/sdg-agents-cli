# Lessons Learned & Research — sdg-agents-cli

> Persistent repository for success patterns, technical research, and architecture decisions.

## Success Patterns

- **Revealing Module Pattern**: All library modules now follow the `const X = {}; export { X }` pattern. This enforces better encapsulation and clear public interfaces compared to inline exports.
- **[S] tasks run single-agent**: Explicit rule in `agent-roles.md` — Planning role executes CODE+TEST directly for `[S]` tasks, skipping the Fast agent spawn to reduce latency and token usage.
- **Selective Lazy Loading**: Injected Knowledge nodes via `Step 0` in `sdg-feat` and `sdg-fix`. This keeps the initial context lean (Token Discipline) while ensuring critical technical memory is available exactly when needed.
- **Pedagogical Writing Soul**: Refined technical documentation to use a calm, inviting, and prose-centric tone. Key constraints include eliminating em dashes and negation-affirmation patterns ("Don't X, Do Y") in favor of descriptive logic and shared vision.
- **Smart Auto-Bump (Changelog Sync)**: Integrated `CHANGELOG.md` promotion into the automated versioning script. The `post-commit` hook now automatically moves entries from `## [Unreleased]` to the newly bumped version header, ensuring the technical narrative stays in sync with `package.json` without manual intervention.
- **Maintainer Self-Sync**: Automatically syncing source-of-truth instructions to the local `.ai/` directory ensures agents stay "charged" with the latest rules without manual intervention.
- **Narrative-First Automation**: When using auto-bump hooks, the developer/agent MUST populate the `[Unreleased]` section of the CHANGELOG before committing, or the version narrative will be lost during the automated promotion.
- **Indentation Preservation & Idempotency**: Implementing `detectIndentation` and `writeJsonAtomic` ensures that CLI-managed file updates (like adding scripts to `package.json`) respect the project's original formatting (2-space, 4-space, or tabs), preventing massive diffs and unnecessary lockfile noise.
- **Token Discipline 2.0 (Mouth vs Soul)**: Integrated **CAVEMAN Full** (chat) and **Pedagogical** (docs) duality. This maximizes technical density in interactions while maintaining humanized, inviting prose in perene project artifacts.
- **Non-Blocking Update Check**: Using internal `fetch` with a short `AbortController` timeout (2s) enables the CLI to detect new versions on npm without impacting startup perceived latency or failing in offline environments.
- **Narrow Bump Sync Alert**: Discovered that automated bump scripts focusing only on `package.json` and `CHANGELOG.md` can cause "context drift" if source assets (e.g., `src/assets/`) are modified but not explicitly staged. The `sdg-end.md` protocol now mandates a final `git status` scan to catch missing assets before push.
- **The Elevation Stack (S0-S3)**: Formalized the rule that dark mode surfaces must become lighter as they "elevate" towards the user. This eliminates "floating darkness" and creates a natural depth-driven hierarchy (S0: Black/Zinc-950 → S3: Zinc-700).
- **Concentric Radius (Anilhamento)**: Implemented the geometric rule `Inner Radius = Outer Radius - Padding` to ensure nested cards and inputs share a common visual center, preventing the "clashing radius" anti-pattern in complex UIs.
- **Naming Governance (Gap de Heurística)**: Identified that naming discipline (avoiding `a, b, i, v`) requires programmatic enforcement that strips comments and strings first. Hardening the `audit` engine to detect single-letter variables prevents structural debt accumulation that conventional linting might miss in complex logic like map/sort parameters.
- **Law 0: THE LAW OF PROTOCOL (Sovereign Gate)**: Established a foundation where project-specific instructions (SSOT) MUST override any general AI training heuristics. This prevents "Protocol Drift" where agents skip mandatory steps like DNA-Checks in favor of speed.
- **DNA-GATE [LOCKED]**: Created a mandatory mental-reset gateway for the CODE phase. The requirement for a specific markdown output block acts as a cognitive trigger, ensuring the agent has purged its training bias before tool execution.
- **Contextual Efficiency (Law 7)**: Integrated project philosophies based on `context-mode`. Lessons: 100/50 line head/tail split (60/40) is optimal for maintaining trace and context without blowing the budget. "Think in Code" (writing analysis scripts) is superior to manual line-by-line reading for large file investigative tasks.
- **Automated Sovereign Audit**: Expanded the `audit` suite to check for Law 0, Law 7, and DNA-GATE strings in the active `.ai` instructions. This ensures the governance locks cannot be silently removed.
- **Visual-Density Fear ≠ Reality (2026-04-18 sweep)**: Exhaustive scan of `src/engine/bin/**` (13 files, ~2,100 LOC) found **zero** double-blank-line violations. `visual-density` WARN stays WARN — promoting to BLOCK without live evidence is cargo-culting; the tier already guards future regressions. Real leakage came from other pattern classes (see next entry).
- **Ceremonial-Void-Return Anti-Pattern**: `explaining-returns` rule pressure produced 9 instances of `const X = console.log(...); return X;` across bin/\*\* — the named const holds `undefined`, satisfying the rule mechanically while destroying its intent. Fix: refine `explaining-returns` to exempt pure side-effect terminators (single-statement `console.log(msg);` with no return is valid), OR add a new `ceremonial-void-return` detector [WARN]. Same false-positive risk profile as inline-assertion-literals (low — void returns are always meaningless).
- **Boolean-Prefix Detector Gap**: Current BLOCK detector matches identifier prefixes but misses boolean-producing RHS expressions (`.match()`, `.test()`, `.includes()`, `array.length > N`). Confirmed hits: `strictMagicMatch` (audit-bundle.mjs:162, used inside `if`), `dryRun` (build-bundle.mjs:246). Tighten detection by scanning RHS call target + usage context (must be used in boolean position to flag).
- **Subagent Noise on Style Rules**: Auditing bin/\*\* via Explore subagent produced ~30 false-positive "explaining-returns violations" on `const X = fn(); return X;` — which is the canonical pattern, not a violation. Always triage subagent style-rule findings against the actual rule text before acting; expect 70-80% noise on narrative/scansion rules.
- **Ceremonial-Literal-Return Pattern (2026-04-18 full-repo sweep)**: After the 3 gates refinement, a grep for `const X = <literal>; return X;` surfaced **~30 sites** across lib/ and bin/ returning atomic values (`null`, `true`/`false`, trivial strings like `'en'`, `'pt-BR'`, `'back'`, `'1 day ago'`). One site (`instruction-assembler.mjs:293-294`) extracts `undefined` into `const injectedResult = undefined; return injectedResult;` — the most vacuous form. Internal inconsistency in `wizard.mjs`: L414 `return 'fullstack';` bare vs L420 `const defaultScope = 'backend'; return defaultScope;` extracted, same function. The `explaining-returns` rule is silent on trivial literals; spirit-of-rule ambiguity needs resolution: either (a) add trivial-literal carve-out to `explaining-returns` (so bare `return null;` / `return 'en';` are OK), or (b) new `ceremonial-literal-return` WARN detector. Recommendation: (a) — carve-out beats new detector because the pattern is a rule-side ambiguity, not a new violation class.
- **Named-Const-Before-Call FP Profile (Data-Access + Read-or-Default)**: The refined `named-const-before-call` as written flags 3 idiomatic patterns that are NOT composition-for-meaning: (1) `path.basename(x)` / `path.join(x, y)` passed to `.replace()` or `fs.*Sync()` — pure data-access primitives; (2) `JSON.parse(fs.readFileSync(x, 'utf8'))` — canonical 2-step read-parse; (3) `fs.existsSync(x) ? fs.readFileSync(x) : ''` — read-or-default ternary. Extracting these adds ceremony without semantic gain. Carve-out proposal: exempt pure data-access call chains (fs/path/JSON.parse) passed to consagrated-API transform targets; keep template-literal/concat/anon-object violations as BLOCK.
- **Gate Regression — `gate-bundle.mjs:21` escaped sanitization**: Last ciclo fixed 11 `ceremonial-void-return` wraps across bin/\*_ but missed `const usageResult = printUsage(); return usageResult;` at [gate-bundle.mjs:21-22](src/engine/bin/maintenance/gate-bundle.mjs#L21-L22). `printUsage` matches the `print_`flagged-target list explicitly. Lesson: sanitization sweeps need post-mutation grep-verify against each gate's flagged-target pattern, not just per-file cursor pass. Validates the`feedback_printer_carveout.md` guidance (printer-pure ≠ caller-of-printer — caller must still avoid ceremonial wrap).

## Research Findings

- **Automated Versioning**: Validated that `npm version` combined with Regex-based `CHANGELOG.md` promotion creates a reliable "Success by Default" experience.
- **Husky Resilience**: Integrated NVM shims in hooks to prevent path-related failures in non-interactive agent shells on Linux.
- **npm `files` + `.npmignore` interaction**: When the `files` field in `package.json` lists a directory, `.npmignore` cannot exclude files inside that directory. npm treats `files` as a strict allowlist.
- **npx symlink resolution**: `process.argv[1]` in an `npx` context holds the symlink path created by npm. To correctly identify the project root or verify if the script is running directly, always use `fs.realpathSync(process.argv[1])` to resolve against `import.meta.url`.
- **Prettier Lint Density**: Discovered 1144 quote-style violations (double vs single). While non-functional, they indicate the importance of early enforcement via CI/CD to prevent technical debt accumulation.
